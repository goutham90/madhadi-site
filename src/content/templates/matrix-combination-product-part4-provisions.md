## Acceptance criteria

- Every applicable added provision maps to a named procedure and a real evidence record.
- Every N/A carries a written rationale.
- The base matches the quality plan, and only the relevant table is used.
- The matrix is approved by QA and kept current as procedures change.

This is the artifact that turns "we comply with Part 211" into a defensible demonstration that the device provisions were each addressed.

## Common inspection findings this matrix prevents

- A provision named in the plan but with no procedure or evidence behind it.
- An N/A with no rationale.
- Both base tables filled in, signaling the team ran two full systems instead of the streamlined approach.

## How to adapt this matrix

1. Set your document number and link the quality plan.
2. Keep only the table that matches your base; delete the other.
3. Map each applicable provision to a real procedure and evidence record.
4. Give a rationale for each N/A and have QA approve.
